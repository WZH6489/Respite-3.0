Section 1 - UI

Glass UI System

We implemented a dynamic visual layer with frosted glass cards, adaptive gradients, and consistent information hierarchy.

Background Engine

Time-aware gradient layers with highlight drift and phase tinting using a dedicated dynamic background view.

Reusable Glass Cards

Shared card modifier with frosted/opaque modes and bordered glass treatment for visual consistency.

Accessible Contrast

Text hierarchy and soft borders preserve readability in both light and dark contexts.

Design Outcome

The app feels unified across Dashboard, Study, Reflections, and Settings while still supporting dense data blocks.

UI Implementation Notes

Core styling came from shared theme primitives and card modifiers, reducing style drift between screens.

Section 2 - Idea

How We Came Up With It

The concept came from one gap: people don't just need blocking, they need guided recovery when attention breaks.

Observed Problem

Standard app timers enforce hard limits but fail at the moment of impulse. Users bounce back into loops with no reflective step.

Core Thesis

Replace static punishment with a behavior loop: detect risk, interrupt intentionally, and convert the moment into a small win.

Design Principle 1

Interventions must be short enough to complete under friction.

Design Principle 2

Every intervention should produce measurable momentum.

Design Principle 3

Insights should guide tomorrow's settings automatically.

Section 3 - Functions

What We Built

Operational features in the current system, from enforcement to insight generation.

Intent Gate

Check-in before opening selected high-risk apps.

Daily Limit

Pause threshold + grace period after intervention completion.

Focus Runtime

Persistent study timer synchronized with live activity state.

Reflections

Prompt-driven logging with archive and editing support.

Recovery Score + Focus Quality

Two distinct behavioral indicators with weekly trend views and metric cards.

Achievements + Streak Momentum

Habit reinforcement via goals, progress bars, and badge milestones.

Extension Architecture

DeviceActivity monitor/report + Shield extensions + Widget extension provide system-level behavior enforcement and surfacing.
